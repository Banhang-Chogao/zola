{# ===== MANIFESTO ===== #}

Design Philosophy · Theme #02

Trust through clarity.
Fintech bold, blog warm.

Z-X lấy cảm hứng từ ZaloPay Blog — sạch sẽ như app fintech, nhưng đủ ấm áp để đọc bài dài. Khác với BrandingX (MoMo pink playful), Z-X thiên về trust + authority qua electric blue + deep navy.

01

Electric Blue trên Cloud White

Brand color không bao giờ trên bg đậm. Luôn blue trên trắng/mist để duy trì contrast + cảm giác "professional fintech".

02

Deep Navy thay Pure Black

Text dùng #0B1834 thay #000 — giảm "hard contrast", tăng cảm giác trust và premium. ZaloPay không bao giờ dùng đen tuyệt đối.

03

Orange chỉ cho Promo

#FF6B00 chỉ xuất hiện ở khuyến mãi, badge "HOT", CTA cuối funnel. Tránh dùng làm primary action — sẽ làm loãng trust signal.

04

Rounded hơn BrandingX

Border-radius card 14px (vs BrandingX 12px). Buttons radius-md 10px. Lift visual softness, gần "app-like" hơn web-like.

Đánh giá tính khả thi switch theme

✅ Khả thi cao

  • Hai theme dùng SCSS variable prefix riêng ($brand-*, $zx-*) → không conflict.
  • Switch qua html[data-theme="zx"] attribute + JS toggle. Apply variable mapping ở root.
  • Layout/spacing/component structure GIỐNG NHAU → chỉ swap màu.

⚠ Thách thức kỹ thuật

  • SCSS variables compile-time → KHÔNG switch runtime được. Cần migrate sang CSS custom properties (--var).
  • SVG inline (logo, icons) màu hardcoded — cần refactor dùng currentColor hoặc CSS var.
  • Image-based assets (hero, thumbnails) khó switch theme. Cần asset cho mỗi theme hoặc dùng CSS filter overlay.
  • Font Manrope vs SF Pro — load thêm font weight tốn extra bandwidth.

💡 Đề xuất implementation

  • Tạo file _themes.scss với :root define BrandingX vars, :root[data-theme="zx"] override Z-X vars.
  • JS toggle + lưu user preference vào localStorage.
  • Add theme switch button vào navbar (góc phải, icon mặt trời/cây).
  • Test contrast WCAG AA cho cả 2 theme trước khi ship.
