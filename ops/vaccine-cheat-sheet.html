🔍 Cách tra cứu nhanh

Bước 1: Gặp lỗi CI → đọc log

Bước 2: Khớp log với "Khi nào dùng" ở bảng → xác định Vaccine ID

Bước 3: Tra cột "File / Test / Detector" → chạy FIXER (KHÔNG chẩn lại)

Bước 4: Commit + push


Ví dụ:

  • Log: 401 Client Error: Repository Not Found HuggingFace → V1
  • Log: API rate limit exceeded Pages → V5
  • Git: mergeable_state: dirty + code OK + data conflict → V10a/V12a
  • QA: 40 internal broken + /bai-N-... link → V14

📌 Operator Notes

• Vaccine có 2 bản (cùng số, khác scope): V10 (CI merge race + compliance heading), V11 (autofixer + taxonomies), V12 (semantic conflict + article depth), V19 (GSC + migration), V22 (editor S-DNA + save).

• Quy tắc bắt buộc: Khi lỗi khớp vaccine dấu hiệu → chạy FIXER ngay (KHÔNG chẩn lại từ đầu). Nếu chưa biết → tra gauge Vaccine library trước khi hỏi ý kiến.

• Process vaccine (V9, V10, V12): Phát hiện thời PR bị conflict / stale base. FIXER là quy trình merge semantic + regenerate, không phải code fix tự động.

• File map: Detector / test / script chính nằm ở cột "File / Test / Detector". Nếu thông tin Unclear → tra CLAUDE.md gốc.

• CI gate: V5 (rate-limit), V10 (dirty), V13 (broken link), V14 (cluster), V25 (route), V29 (smoke) có thể chặn auto-merge. Phải fix trước push.
